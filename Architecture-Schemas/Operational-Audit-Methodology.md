# Operational Architecture Audit — Methodology

> "Do not start building until you have finished mapping. Building on an unmapped system embeds the dysfunction into the solution."

---

## Purpose

The Operational Architecture Audit is a structured, end-to-end investigation of how a business actually moves — before any system is built, any tool is purchased, or any automation is installed.

It maps every process, handoff, and decision point with one goal: to produce a clear picture of where work flows well, where it breaks, and what to build first.

**This is not a workshop. It is a forensic investigation.**

---

## Who This Is For

- DTC / ecommerce brands running on Founder/Owner memory
- Companies with 3+ sales channels, multiple fulfilment points, or international production
- Businesses where the “system” lives in someone’s head
- Founders who know something is wrong but can’t name it precisely

---

## Phase 1 — Discovery (Days 1–3)

- Shadow a live operational cycle (order window, production kick-off, dispatch run)
- Interview department leads on current process — not the ideal process, the actual one
- Inventory all active spreadsheets, tools, and comms channels
- Map every place where data lives and who owns it

**Key question:** Where does work go when no one is looking?

---

## Phase 2 — Mapping (Days 4–8)

- Build end-to-end Order-to-Dispatch flow diagram
- Identify every handoff, decision point, and manual step
- Flag: data duplication, trust gaps, missing ownership, manual bridges between systems

Typical DTC flow mapped:
Order window open → Shopify export → MTO extraction → stock check → allocation → factory PO → production tracking → inbound shipment → fulfilment → dispatch → returns & exchanges

---

## Phase 3 — Synthesis (Days 9–14)

- Define Single Source of Truth: entities, statuses, vocabulary
- Write 30/60/90 Day Roadmap: what to build first / defer / ignore
- Deliver Escalation Matrix v1: what needs Founder/Owner vs what can be delegated
- Build Tracker Skeleton: live prototype mirroring real workflow

---

## Deliverables

1. Written audit document: workflow map, gap analysis, findings
2. Tracker skeleton (Notion or Airtable) with agreed statuses, ownership, naming conventions
3. 30/60/90 Roadmap with tooling recommendation and cost range
4. Escalation Matrix v1

---

## Acceptance Criteria

- End-to-end workflow documented clearly enough for a new operator to follow without the Founder/Owner
- Live tracker skeleton with agreed statuses, ownership, naming
- Roadmap written, prioritised, and signed off

---

## System: Before → After

| Dimension | Before (Day 0) | After (Day 90) |
|---|---|---|
| Order visibility | Founder/Owner is the system | Live tracker visible to all departments |
| Information flow | Each department runs on different data | Single Source of Truth, one vocabulary |
| Handoffs | Verbal, email threads, messaging apps | Status-driven, documented, traceable |
| Founder involvement | Every question routes to Founder/Owner | Escalation matrix defines what needs them |
| Bottlenecks | Manual bridges between every system | Structured handoffs with clear triggers |
| Audit trail | Reconstructed after the fact | Real-time status + history |
| New hire onboarding | Months of tribal knowledge absorption | Documented workflow, followable day one |
